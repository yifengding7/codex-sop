## 命题人视角：命题逻辑与结构对齐
### 关键结论
1. **考纲 → 结构映射缺失**：A 链首屏强调 WHY/HOW/RESULT，但没有显式标注考纲能力与题型，难以证明命题意图。【F:Project_Instructions.md†L5-L32】【F:templates/A4_Template.md†L9-L37】
2. **题型图示判据缺位**：type_adapter 仅列检查项，无“题型 × 图示选型”约束，存在使用非核心图示风险。【F:type_adapters.yaml†L3-L60】
3. **干扰项未被验证**：项目质量门聚焦字数/结构，无“干扰项/常错路径”检查，无法证明题目严密性。【F:PROJECT_README.md†L3-L20】【F:validators/validator.py†L24-L118】
4. **得分点与命题依据断档**：得分点列表虽标 P#，却缺少“命题依据/考纲条款”列，无法回溯。【F:templates/A4_Template.md†L9-L37】

### 改进抓手
- **模板升级**：
  - 在首屏 A 链标题区新增“考纲能力代码 + 题型”字段，并要求列出“核心考查点/次级考查点”。
  - 在“得分点清单”中新增“考纲出处/教材章节”列，validator 验证非空并校验格式。
- **图示规范体系**：基于 type_adapter 输出《题型 × 图示选型矩阵》，包括“必备元素、可选元素、禁用图示、反例库”，并在模板中引用。
- **干扰项校验**：在 README 增补“干扰项描述”要求；validator 新增检查：若题型为选择题，必须包含“干扰项诊断”段落，且覆盖 ≥2 条常见错误路径。【F:PROJECT_README.md†L3-L20】【F:validators/validator.py†L61-L118】

### 验收与风险缓释
- 每篇稿件生成《考纲映射表》留档，供命题联合评审。
- 抽检 5 篇压轴题，确认图示选型符合矩阵；若不符，回滚模板。
- 校验 3 套选择题确保“干扰项诊断”段落完整，评估学生复现率。

## 顶级辅导教练视角：执行力与评估闭环
### 关键结论
1. **错因→纠偏链路缺位**：现有 validator 只查字段存在，不要求“错因、纠偏动作、预估时长”，教练需二次加工。【F:validators/validator.py†L8-L152】【F:templates/A4_Template.md†L20-L31】
2. **术语旁注覆盖率不可见**：Glossary CSV 提供标准旁注，但无自动比对，批量课件质量无法量化。【F:PROJECT_README.md†L7-L13】【F:glossary/Terms_MATH.csv†L1-L11】
3. **B/C 方案缺少量化准入**：只有“等价更快/≤3 点”文字描述，无“节省步数/时长”等硬指标。【F:Project_Instructions.md†L12-L33】【F:templates/A4_Template.md†L20-L24】
4. **题型自查未自动注入**：type_adapter 中的题型条目未在模板或 validator 中强制显示，教练需人工核对。【F:type_adapters.yaml†L3-L50】【F:templates/A4_Template.md†L26-L31】

### 改进抓手
- **错因纠偏表**：在模板 A 链下方新增“错因 → 纠偏动作 → 练习时长/次数”表格，validator 校验至少列出 3 条，且包含“纠偏完成标准”。
- **术语覆盖报告**：开发脚本读取终稿，统计 glossary 术语的旁注命中率；将报告接入质量门，未达 95% 则退回。
- **量化 B/C 方案**：在模板中加入“原方案步数/时长/得分率”与“B/C 方案改进值”字段，validator 验证填写且满足阈值（≥30% 步数缩减或 ≥40 秒节省）。
- **题型自查自动化**：根据 type_adapter 在模板内动态渲染题型特有自查项，并在 validator 中检查对应条目是否完成。

### 验收与风险缓释
- 每周输出《错因纠偏命中率》报表；若命中率 <80%，回溯训练脚本。
- 在教练试用群内进行 2 周小样验证，确认量化字段能支持训练排班。

## 错题学生视角：认知负荷与自主演练
### 关键结论
1. **首屏信息超载**：WHY/HOW/RESULT、等式、P#、最简写法叠加，弱生缺少分层引导，易在首屏失焦。【F:Project_Instructions.md†L5-L16】
2. **自查卡不可操作**：仅给出 4 条检查项，没有“合格/不合格样稿”或“失败返回路径”，学生难以自评。【F:templates/A4_Template.md†L26-L31】
3. **复盘链未闭环**：回忆链、锚点、微练写在说明中，但模板缺少错题记录与复练任务的结构化输出。【F:Project_Instructions.md†L12-L35】【F:templates/A4_Template.md†L38-L51】
4. **术语旁注无层级**：Glossary 未区分初学/熟练/竞赛级别旁注，弱生需要再次查阅外部资料。【F:glossary/Terms_MATH.csv†L1-L11】

### 改进抓手
- **分层首屏**：新增“起始提示卡”，按照保底/标准/冲刺 3 档逐步展开，允许弱基学生先读保底版，再点开细节。模板使用折叠块指引。
- **自查样稿 + 反馈语**：提供“30 秒自查”合格与不合格样稿，并在 validator 中检测是否包含“失败 → 返回 P#”类反馈句。
- **错题复盘工单**：设计附录表格（触发条件、复写要求、微练题链接、复练频率），与回忆链字段互相引用，形成“练→查→复练”闭环。
- **术语层级标记**：在 glossary 中增加“难度级别/展开策略”列，并在模板要求勾选对应级别，降低额外检索成本。

### 验收与风险缓释
- 与 3 名弱基学生进行可用性测试，指标：① 首屏 1 分钟内能说出目标；② 30 秒内完成自查并知道下一步。
- 监控错题工单使用率，确保 70% 以上错题被录入并触发复练任务。

## 跨角色落地路线图
1. **第 0 周：规范发布**
   - 发布新版模板、validator、题型图示矩阵与示例稿；同步 README 中的干扰项要求。
2. **第 1–2 周：灰度验证**
   - 选择 3 套不同题型试点，记录考纲映射、错因纠偏表填写情况，收集教练与学生反馈。
   - 产出术语覆盖率与自查命中率报告，评估自动化脚本稳定性。
3. **第 3 周：全面上线与培训**
   - 对命题人、教练、学生运营分别举行培训：强调考纲映射、量化 B/C 方案、分层首屏使用方法。
   - 将错题复盘工单接入学生运营的日常复练流程，追踪指标仪表盘。
4. **持续迭代**
   - 每月召开跨角色复盘，更新题型图示反例库与弱生体验包。
   - 根据数据反馈迭代 validator 规则（如增加“课堂讲评脚本”字段）。
